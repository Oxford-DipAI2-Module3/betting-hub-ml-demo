🤖 ML Models Explained

1. LightGBM (Active Production Model)

Best for: Production deployment

Why we chose it:

  • Perfect balance of speed (20ms) and accuracy (99.93% R²)
  • Handles categorical features natively (no manual encoding needed)
  • Memory efficient - can run on standard servers
  • Strong personalization support

Training Configuration:

model = lgb.LGBMRegressor(
    n_estimators=100,
    max_depth=6,
    learning_rate=0.1,
    num_leaves=31
)

# Training data: 322,688 clicks
# Features: 47 (10 categorical + 37 numerical)
# Train/test split: 80/20

2. XGBoost (Highest Accuracy)

Best for: Maximum accuracy (when version issues resolved)

Performance:

  • R²: 99.95% (0.02% better than LightGBM)
  • Fastest training: 1.19s
  • Excellent personalization handling
  • Industry standard for tabular data

3. Random Forest

Best for: Stability and robustness

  • R²: 99.56% (still excellent)
  • No hyperparameter tuning needed
  • Very stable predictions (low variance)
  • Trade-off: 23x slower training, 5x slower inference

4. Ridge Regression

Best for: Speed and interpretability

  • R²: 96.75% (acceptable for linear model)
  • Ultra-fast: 0.14s training, <5ms inference
  • Interpretable coefficients
  • Good baseline, but can't capture complex interactions

5. ElasticNet (Failed)

Status: Not recommended

  • R²: -0.03% (worse than predicting mean)
  • L1 regularization too aggressive - eliminated important features
  • Not suitable for this dataset

🔧 Feature Engineering

Feature Categories (47 Total)

1. Tile Features (15 features)

  • tile_position - Array index (1-30)
  • bet_type - Acca, Treble, Double, Mega Acca, etc.
  • bookmaker - Dabble, Betway, SBK, etc.
  • sport - Football, Racing, etc.
  • tile_type - Bet, Promo, NCO, Feature Shortcut
  • odds_min, odds_max, odds_mid - Parsed from "3/1 - 11/2"
  • is_acca, is_mega_acca, is_treble - Binary flags
  • is_boost, is_builder, is_rab - Binary flags
  • is_top_bookmaker - Premium bookmaker flag

2. Position Features (5 features)

  • position_bucket - top5, top10, top15, top20, below20
  • odds_category - very_short, short, medium, long, very_long

3. Time Features (7 features)

  • hour_of_day - 0-23
  • is_morning, is_lunch, is_afternoon, is_evening, is_night
  • is_weekend - Saturday/Sunday flag

4. User Features (20 features)

  • user_experience - log(session_number)
  • user_preferred_bet_type - Most clicked bet type
  • user_preferred_bookmaker - Most clicked bookmaker
  • user_preferred_sport - Most clicked sport
  • user_acca_rate - % of clicks on accas
  • user_mega_acca_rate - % of clicks on mega accas
  • user_boost_rate - % of clicks on boost tiles
  • user_avg_odds - Average odds of clicked tiles
  • matches_user_bet_type - Does tile match user preference?
  • matches_user_bookmaker - Does tile match user preference?
  • user_is_power_user - 50+ sessions
  • user_loves_accas - Acca rate > 60%
  • user_engagement_score - Composite engagement metric

👤 Personalization Logic

Cold Start Problem

For new users with no history:

  • Position weight: 54% (maximum)
  • Bookmaker/bet type: Standard weights
  • Personalization boost: 0%
  • Strategy: Show most popular tiles first

Casual Users (5-20 sessions)

# Example: User prefers Dabble Accas
if tile.bet_type == "Acca" and tile.bookmaker == "Dabble":
    score += 0.15  # 15% boost
    
if tile.tile_type == "NCO" and tile.bookmaker == "Dabble":
    score -= 0.50  # 50% penalty - already signed up

Power Users (50+ sessions)

# Example: User loves Betway Mega Accas
personalization_boost = 0

# Exact bet type match
if tile.bet_type == user.preferred_bet_type:
    personalization_boost += 0.25  # HUGE boost

# Exact bookmaker match (but not NCOs)
if tile.bookmaker == user.preferred_bookmaker:
    if tile.tile_type != "NCO":
        personalization_boost += 0.20
        
# Both match = JACKPOT
if both_match:
    personalization_boost += 0.10  # Bonus!

# Demote NCOs from signed-up bookmakers
if tile.tile_type == "NCO" and tile.bookmaker in user.signed_up:
    personalization_boost -= 0.50  # Massive penalty

Bookmaker Signup Tracking

Key Innovation: Track which bookmakers users have already signed up with.

How it works:

  1. Track when users click NCO tiles
  2. Store list of bookmakers per user
  3. Heavily demote NCOs from those bookmakers (-50% score)
  4. Promote NCOs from new bookmakers

Impact: Prevents wasting premium positions on unusable signup offers.

  2. Firebase → BigQuery: Events flow automatically
  3. Daily ETL: Calculate user profiles (preferences, signups)
  4. Weekly Retraining: Model updates with new data
  5. A/B Testing: Compare ML vs current ranking
  6. Validation: Measure actual CTR improvement

User Profile Query (BigQuery)

CREATE OR REPLACE TABLE user_profiles AS
WITH clicks AS (
  SELECT 
    user_id,
    bookmaker,
    bet_type,
    tile_type,
    COUNT(*) as clicks
  FROM tile_clicks
  WHERE date >= DATE_SUB(CURRENT_DATE(), INTERVAL 90 DAYS)
  GROUP BY user_id, bookmaker, bet_type, tile_type
)
SELECT
  user_id,
  ARRAY_AGG(bookmaker ORDER BY clicks DESC LIMIT 1)[OFFSET(0)] as preferred_bookmaker,
  ARRAY_AGG(bet_type ORDER BY clicks DESC LIMIT 1)[OFFSET(0)] as preferred_bet_type,
  ARRAY_AGG(DISTINCT bookmaker IGNORE NULLS) 
    FILTER(WHERE tile_type = 'NCO') as signed_up_bookmakers,
  SUM(clicks) as total_clicks
FROM clicks
GROUP BY user_id;

  4. Monitor response times (should be <50ms)
  5. Enable impression tracking in app

Phase 2: A/B Testing (Week 3-6)

// Pseudocode for A/B test
function getTiles(userId) {
  const tiles = fetchFromCheckd();
  
  if (isInMLGroup(userId)) {
    // 50% of users - ML ranking
    return rankWithML(tiles, userId);
  } else {
    // 50% of users - current ranking
    return tiles;
  }
}

Troubleshooting

Common Issues:

  • Slow Response: Check model loading (should load once at startup)
  • Wrong Positions: Verify using array index, not "order" field
  • CTR Not Improving: Check impression tracking is working
  • Model Errors: Verify all 47 features present
